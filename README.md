# Análisis de Eficiencia de Algoritmos de Ordenamiento

## Objetivo

Comparar el rendimiento entre el algoritmo Bubble Sort y el método nativo `sorted()` de Python, aplicando análisis asintótico y mediciones de tiempo.

## Algoritmos implementados

- **Bubble Sort**: Algoritmo de ordenamiento simple con complejidad O(n²).
- **sorted()**: Método optimizado de Python basado en Timsort, con complejidad O(n log n).

## Metodología

Se generaron listas de números aleatorios con tamaños: 100, 500, 1000 y 5000 elementos. Se midió el tiempo de ejecución de cada algoritmo usando `time.perf_counter()`.

Para esto se cargara un excel con la tabla a este proyecto.

## Análisis

- **Bubble Sort** muestra un crecimiento cuadrático en tiempo de ejecución.
- **sorted()** es significativamente más rápido y escalable.
- Se observa un claro trade-off entre simplicidad y eficiencia.

## Conclusión

Para listas grandes, `sorted()` es la opción recomendada por su rendimiento superior. Bubble Sort es útil para fines didácticos pero no práctico en producción.
